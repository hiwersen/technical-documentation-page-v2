INTRODUCTION

Web forms serve as powerful tools for user interaction, primarily collecting data and enabling user interface control. However, their full potential can be elusive due to technical complexities in implementation.

This guide explores key web form concepts, including:

  • HTML structural markup
  • Control styling techniques
  • Data validation methods
  • Server submission processes

Prerequisites

Familiarize yourself with the Introduction to HTML before proceeding.

However, form elements exceed the complexity of standard HTML components, requiring integrated CSS and JavaScript knowledge to maximize their effectiveness.

Design

For optimal user experience (UX), remember that form length directly impacts abandonment rates. Maintain simplicity and focus by requesting only essential information.

Our demonstration includes three text fields (name, email, message) and a submission button that transmits data to the server when clicked.

The form element

The <form> element functions as a specialized container similar to section or footer elements, but specifically designed for forms; it also features unique attributes that configure form behavior.

While all its attributes are optional, best practice dictates setting at minimum both action and method attributes:

  • The action attribute specifies the destination URL where collected form data should be sent upon submission.
  • The method attribute determines which HTTP method transmits the data (typically get or post).

The label, input and textarea elements

Note that each label element includes a for attribute containing the id of its associated form control — this creates the connection between label and control, allowing users with mouse, trackpad, or touch devices to activate the control by clicking its label, while simultaneously providing an accessible name for screen readers to announce to their users.

Our contact form features three text fields, each paired with a corresponding <label>:

  • The name field uses an <input> of type text: a single-line text field.
  • The email field uses an input of type email: a single-line text field that validates email address formats.
  • The message field implements a <textarea>: a multiline text field.

The type attribute is the most important attribute for input elements, fundamentally determining both their appearance and behavior.

  • The default value for this attribute is "text", as utilized in the first input example — this default setting creates a standard single-line field accepting any text content.
  • The value "email", implemented in the second input, generates a single-line field that validates for properly formatted email addresses.

The button element

The <button> element enables users to send or "submit" their information after completing the form.

The button element supports a type attribute that accepts three possible values: submit, reset, or button:

  • Clicking a submit button (the default type) transmits the form data to the webpage specified in the form element's action attribute.
  • Clicking a reset button immediately restores all form widgets to their default values. This is generally considered poor UX practice, so avoid using this type unless specifically necessary.
  • Clicking a button of type button performs no default action. While this seems counterintuitive, it's extremely valuable for implementing custom functionality via JavaScript.

  /* Center the form on the page */

  border: 1px solid #ccc;

input,

  /* Match form fields' properties */
  font: 1em sans-serif;
  width: 300px;
  box-sizing: border-box;
  border: 1px solid #999;
}
                  
textarea {
  /* Align multiline text fields
    with their labels */
  vertical-align: top;
}
                  
button {
  /* This extra margin represents 
    roughly the same space as the space
    between the labels and their text fields */
  margin-left: 98px;
}
                  
input:not(input:first-of-type),
label:not(label:first-of-type),
textarea,
button {

Submitting data

The form element determines where and how data is transmitted to the server through its action and method attributes.

Form data is transmitted as name/value pairs. Each form control requires a name attribute. These names serve dual purposes: they tell browsers how to label each data piece and enable servers to process individual data items by name.

Our example transmits three data pieces labeled "user-name", "user-email", and "user-message". This data will be sent to "/my-handling-form-page" using the HTTP POST method.

On the server side, the script at "/my-handling-form-page" receives the data as three key/value pairs within the HTTP request. The specific handling of this data depends on your implementation and exceeds this guide's scope.
